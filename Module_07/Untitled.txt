Assignment 7 

Assignment: Building a Parallel Version of Quicksort

In this assignment, you will implement a parallel version of Quicksort in Rust using only the standard library.

You will:

build a correct sequential Quicksort,
extend it into a parallel divide-and-conquer algorithm,
use Rust’s ownership system and scoped threads to ensure safety,
evaluate performance and explain tradeoffs.
⚠️ You may NOT use external crates such as Rayon.

Learning Objectives

By completing this assignment, you should be able to:

implement recursive algorithms in idiomatic Rust,
safely use parallelism with std::thread::scope,
reason about task granularity and cutoff thresholds,
explain when parallelism helps (and when it hurts),
write and analyze performance experiments.
Assignment Requirements

Part 1 — Sequential Quicksort

Implement an in-place Quicksort:

pub fn sequential_quicksort<T: Ord>(data: &mut [T])
Requirements

Must sort correctly for all inputs

Must use in-place partitioning

Must handle:

empty slices
already sorted data
reverse sorted data
duplicate-heavy data
Part 2 — Parallel Quicksort

Implement:

pub fn parallel_quicksort<T: Ord + Send>(data: &mut [T], cutoff: usize)
Requirements

Use divide-and-conquer recursion

Use std::thread::scope

After partitioning:

sort left and right halves in parallel (when large enough)
Use a cutoff threshold

below cutoff → sequential version
Avoid excessive thread creation

Required Rust Concepts

1. Disjoint Mutable Slices

You will need:

let (left, right) = data.split_at_mut(mid);
This allows safe parallel mutation.

2. Scoped Threads

Use:

std::thread::scope(|s| {
    s.spawn(|| { /* work */ });
});
This allows borrowing slices safely across threads.

3. Avoid Shared State

❌ Do NOT use:

Mutex
Arc
shared global data
Your algorithm should rely on non-overlapping slices.

Design Expectations

Your parallel algorithm should:

Partition the slice
Split into two disjoint regions
Decide whether to parallelize
Spawn at most one thread per recursion level
Recurse on both halves
